# Publishing to Chrome Web Store - Guide

This guide will walk you through publishing the ABUGO Intercom Actions extension to the Chrome Web Store for easy distribution to your team.

## Prerequisites

1. **Google Account** - Use your work Google account (e.g., @abugo.com)
2. **$5 One-time Fee** - Chrome Web Store developer registration fee
3. **Extension Files** - All files ready in the `intercom` folder

## Step 1: Register as a Chrome Web Store Developer

1. Go to [Chrome Web Store Developer Dashboard](https://chrome.google.com/webstore/devconsole)
2. Sign in with your Google account
3. Pay the **$5 one-time registration fee**
4. Accept the Developer Agreement

**Note:** This fee is one-time and covers unlimited extensions.

## Step 2: Prepare Your Extension

### 2.1 Check Manifest Version

Ensure your `manifest.json` has:
- Proper version number (e.g., `"version": "1.0.0"`)
- Complete description
- All required permissions

### 2.2 Create Icons (Optional but Recommended)

Create icon files for the store:
- `icon16.png` - 16x16 pixels
- `icon48.png` - 48x48 pixels  
- `icon128.png` - 128x128 pixels

**Quick option:** You can use online tools or create simple icons. The extension will work without them, but they improve the store listing.

### 2.3 Create Zip File for Upload

Create a zip file with **only** the files needed for the extension:

```bash
cd extensions/intercom
zip -r intercom-store.zip . -x "*.pem" "*.crx" "*.zip" "*.git*" "README.md" "INSTALLATION.md" "PUBLISHING.md" "SHARING.md"
```

**Files to include:**
- ‚úÖ `manifest.json`
- ‚úÖ `content.js`
- ‚úÖ `icon16.png`, `icon48.png`, `icon128.png` (if you have them)

**Files to exclude:**
- ‚ùå `.pem` files (private keys)
- ‚ùå `.crx` files (will be generated by Chrome)
- ‚ùå Documentation files (README, INSTALLATION, etc.)
- ‚ùå `.git` files

## Step 3: Upload to Chrome Web Store

1. Go to [Chrome Web Store Developer Dashboard](https://chrome.google.com/webstore/devconsole)
2. Click **"New Item"** or **"Add new item"**
3. Click **"Choose file"** and select your `intercom-store.zip` file
4. Click **"Upload"**
5. Wait for Chrome to process and validate your extension

## Step 4: Fill Out Store Listing

### Required Information

**Name:**
```
ABUGO Intercom Actions
```

**Summary (132 characters max):**
```
Adds Actions section to Intercom Knowledge Hub for article translation workflow
```

**Description:**
```
This extension adds an "Actions" section to the Intercom Knowledge Hub sidebar, allowing you to trigger translation workflows for articles.

Features:
- Adds "Actions" section between "Data" and "Fin" in the sidebar
- "Translate to all languages" button for English articles
- Automatic article ID detection
- Language-aware (only enabled for English articles)

For internal ABUGO use only.
```

**Category:**
- Select **"Productivity"** or **"Developer Tools"**

**Language:**
- Select **"English"**

### Privacy Settings (Important!)

**Visibility Options:**

1. **Unlisted** (Recommended for internal use)
   - Extension is not searchable in the store
   - Only accessible via direct link
   - Share the link with your team
   - ‚úÖ Best for internal distribution

2. **Private** (Most secure)
   - Only accessible to specific Google accounts
   - Requires adding each user's email
   - More setup but most secure

3. **Public** (Not recommended)
   - Anyone can find and install it
   - ‚ùå Not suitable for internal tools

**Recommendation:** Choose **"Unlisted"** - you can share the direct link with your team.

### Additional Details

**Screenshots:**
- Take 1-2 screenshots showing the extension in action
- Recommended sizes: 1280x800 or 640x400
- Show the "Actions" section in the Intercom sidebar

**Privacy Practices:**
- Select: "This extension handles user data"
- Explain: "Sends article ID to internal webhook for translation processing"

**Single Purpose:**
- Yes, this extension has a single purpose

## Step 5: Submit for Review

1. Review all information
2. Click **"Submit for Review"**
3. Chrome will review your extension (usually 1-3 business days)
4. You'll receive an email when it's approved

**Review Criteria:**
- Extension works as described
- No malicious code
- Privacy practices are clear
- Follows Chrome Web Store policies

## Step 6: Share with Your Team

Once approved:

1. Copy the extension's store URL (looks like: `https://chrome.google.com/webstore/detail/...`)
2. Share the link with your team in Slack: **#abugo_ai_automation_team**
3. Teammates can install with one click - no developer mode needed!

## Updating the Extension

**Important:** Always bump the version number when making changes!

When you need to update:

1. **Bump the version number** in `manifest.json`:
   - Patch version (1.0.0 ‚Üí 1.0.1) for bug fixes and small changes
   - Minor version (1.0.0 ‚Üí 1.1.0) for new features
   - Major version (1.0.0 ‚Üí 2.0.0) for breaking changes
   
   ```json
   "version": "1.0.1"
   ```

2. Create a new zip file (same process as Step 2.3)

3. Go to [Developer Dashboard](https://chrome.google.com/webstore/devconsole)

4. Find your extension and click **"Edit"**

5. Click **"Upload new package"** and select the new zip

6. Click **"Submit for update"**

7. Updates are usually reviewed faster (often same day)

8. Users will automatically get the update within 24 hours

## Troubleshooting

**Extension rejected?**
- Check the email for specific reasons
- Common issues: missing description, unclear privacy practices
- Fix and resubmit

**Need to make it more private?**
- Change visibility from "Unlisted" to "Private"
- Add specific Google account emails

**Questions?**
